Программу можно проверить на корректность сборки на тестовой системе.
Для этого потребуется установленный Docker. Если на системе есть Docker, то можно зайти в директорию materials/build/Docker и запустить оттуда скрипт run.sh. 
Скрипт обернет ваше решение в докер и запустит его вместе с типовым сценарием сборки для проверки работоспособности бэкэнда и отсутствия утечек.
